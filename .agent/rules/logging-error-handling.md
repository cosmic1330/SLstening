# 日誌與錯誤處理規範

## 內容描述
規範跨環境（Tauri/Rust/React）的日誌記錄與異常捕捉機制，確保桌面端應用的可維護性。

## 核心規則
1. **統一日誌通道**：
   - 前端應透過專用的 Rust Command 將關鍵操作日誌傳回後端。
   - 所有日誌應分類記錄（Info, Warn, Error）並存儲於應用數據目錄下的 `logs/` 資料夾中。
2. **前端異常捕捉**：
   - 必須使用 `ErrorBoundary` 包裹主要頁面與關鍵功能模組，防止單一元件崩潰導致整個應用白屏。
   - 未捕獲的 Promise Rejection 必須有全局監聽。
3. **後端 Panic 處理**：
   - Rust 代碼中禁止濫用 `panic!`。
   - 在關鍵流程中應使用封裝良好的錯誤處理框架，整合進 Tauri 的錯誤傳遞機制，確保錯誤能以人類可讀的形式顯示給用戶。
4. **日誌隱私**：
   - 嚴禁在日誌中記錄用戶的敏感資訊（如 Supabase 密碼、個人交易明細），僅紀錄技術錯誤與功能流轉。
