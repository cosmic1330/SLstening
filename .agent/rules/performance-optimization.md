# 效能優化規範 (視覺化與運算)

## 內容描述
針對股票分析應用的數據密集型特點，規範圖表渲染與指標計算的效能優化路徑。

## 核心規則
1. **圖表渲染限制 (Charts)**：
   - 使用 Recharts 渲染 K 線圖或技術指標時，單次渲染的數據點建議控制在 **300 - 500 點** 以內。
   - 超過此限量的歷史數據應使用滑動視窗 (Sliding Window) 或虛擬滾動 (Virtual Rendering) 技術處理。
2. **指標計算緩存 (Computing)**：
   - 在前端計算技術指標（如 MA, EMA, KD, OBV）時，必須使用 `useMemo` 或專門的緩存服務。
   - 禁止在 React 組件的主體內執行大規模數據遍歷運算。
3. **計算分流 (Offloading)**：
   - 複雜的技術篩選邏輯（如 Schoice 的多條件查詢）必須在 Rust 層級執行，利用 Rust 的高效運算能力並避免阻塞前端渲染線程。
4. **防抖與節流 (Debounce & Throttle)**：
   - 針對視窗大小調整 (Resize) 或圖表縮放 (Zoom) 等高頻事件之觸發，必須實作防抖或節流處理。
