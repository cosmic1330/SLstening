# UI/UX Concept Design

## 1️⃣ Purpose & Scope

Ensure every UI/UX concept is **user-centered, clear, and feasible** before moving into high-fidelity design or development.

- Prevents vague ideas
- Enforces strong user empathy
- Blocks concepts that cannot be realistically implemented

---

## 2️⃣ Pre-Requisites

You must have:

- Defined user personas or target audience
- Core product goal / problem statement
- Basic understanding of platform (Web / Mobile / Desktop)
- Any existing brand guidelines (if applicable)

### Concept Quality Checklist

A solid concept must include:

- Clear user problem or opportunity
- Single primary user flow
- Key user actions
- Emotional tone / brand feeling
- High-level success criteria

---

### 3️⃣ Concept Lock (Hard Gate)

Before drawing any wireframes or moodboards, you MUST:

- Present the **final concept summary**
- Specify:
  - Target user persona
  - Primary user goal
  - Core user journey (max 5 steps)
  - Desired emotional experience
  - Key screens / touchpoints

Ask explicitly:

> “Is this the final concept we are committing to for this UI/UX design?”

**Do NOT proceed until confirmed.**

---

### 4️⃣ Assumptions & Validity Check (Mandatory)

Explicitly list assumptions about:

- User behavior and mental model
- Device usage context
- Technical constraints (API, performance, etc.)
- Accessibility needs
- Competitive benchmarks

If assumptions are weak or missing:
- Warn the user
- Recommend user research or clarification before continuing

---

### 5️⃣ Concept Direction Selection

Choose the simplest valid direction:

- **Minimal & Clean** – focus on clarity and speed
- **Dashboard-First** – data-heavy monitoring style
- **Guided Workflow** – step-by-step user journey
- **Exploratory & Rich** – feature discovery oriented

Default to **Minimal & Clean** unless there is a clear reason otherwise.

---

### 6️⃣ Concept Elements Definition

#### Primary Element (Mandatory)

- Main user flow / key screen
- Directly tied to the core goal
- Pre-defined and frozen before sketching

#### Secondary Elements

- Supporting screens
- Navigation pattern
- Micro-interactions (high-level only)
- Information hierarchy

#### Guardrail Elements

- Accessibility compliance (WCAG basics)
- Performance considerations
- Brand consistency
- Simplicity rule (no more than 3 actions per screen)

---

### 7️⃣ Concept Scope & Feasibility Estimate

Define upfront:

- Number of key screens (recommended ≤ 8 for concept phase)
- Core user journey length
- Estimated complexity level
- Any technical red flags

**Do NOT proceed without a realistic scope estimate.**

---

### 8️⃣ Concept Readiness Gate (Hard Stop)

You may proceed to wireframing / low-fidelity prototyping **only if all are true**:

- Concept is locked
- Primary user flow is frozen
- Assumptions are validated
- Scope is clearly defined
- Guardrails are respected

If any item is missing, stop and resolve it.

---

## Running the Concept Phase

### During Concept Creation

**DO:**

- Focus on user goals and flows
- Use simple sketches / low-fi wireframes
- Think in user stories

**DO NOT:**

- Jump into visual styling or colors
- Add detailed animations
- Design every possible edge case
- Change core concept mid-process

---

## Reviewing the Concept

### Evaluation Discipline

When reviewing:

- Do NOT judge by beauty
- Do NOT add features not in the locked concept
- Separate user value from designer preference

### Interpretation Outcomes

| Result                  | Action                                      |
|-------------------------|---------------------------------------------|
| Clear & user-focused    | Move to low-fidelity wireframes             |
| Confusing flow          | Redefine primary user journey               |
| Scope too large         | Cut non-essential screens                   |
| Guardrail violation     | Fix immediately before proceeding           |

---

## Documentation & Learning

### Concept Record (Mandatory)

Document:

- Locked concept summary
- User journey map
- Key screens (low-fi)
- Assumptions & decisions
- Learnings
- Next steps (wireframe / user testing)

Store records in a shared, searchable location.

---

## Refusal Conditions (Safety)

Refuse to proceed if:

- User persona or goal is undefined
- Primary user flow cannot be clearly stated
- Scope is vague or unlimited
- No validation of assumptions
- Multiple conflicting concepts without prioritization

Explain why and recommend next steps.

---

## Key Principles (Non-Negotiable)

- User first, visuals second
- One concept at a time
- Clarity over creativity
- Simplicity is strength
- Concept must be explainable in 60 seconds
- Lock before you draw

---

## Final Reminder

UI/UX concept design is not about making it look good.  
It is about **making sure we are designing the right thing** for the right user.

If you feel tempted to add more features, fancy interactions, or “just one more screen” —  
that is the signal to **slow down and re-check the concept**.
