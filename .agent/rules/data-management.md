# 數據管理與同步規範

## 內容描述
規範應用程式中 SQLite 本地數據與 Supabase 雲端數據的持久化、同步及遷移邏輯。

## 核心規則
1. **Offline-First 策略**：
   - UI 元件必須優先從本地 SQLite 讀取數據以確保即時響應。
   - 數據更新應由 Rust 後端在背景異步執行，完成後透過 `emit` 事件通知前端重新讀取。
2. **數據同步邏輯**：
   - **自選股與設定**：應在用戶登入時、應用啟動時及手動觸發時，將本地 SQLite 與 Supabase 進行雙向同步（以最新時間戳為準）。
   - **股票交易數據**：僅儲存於本地 SQLite 作為緩存，不應同步至雲端以節省頻寬與儲存空間。
3. **數據庫遷移 (Migration)**：
   - 所有對 SQLite Schema 的變更必須在 `src-tauri` 中有對應的遷移邏輯（如 `migrations/*.sql`）。
   - 禁止在沒有遷移指令的情況下直接修改本地資料庫結構。
4. **一致性保證**：
   - 涉及到多個資料表的寫入操作，必須使用資料庫事務 (Transaction) 確保原子性。
